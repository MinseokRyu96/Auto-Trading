# 트레이딩 로직 리서치 요약

## 결론

초기 자동매매는 초단타가 아니라 **유동성 필터 + 중기 모멘텀 + 변동성 제어 + 대형주 편향 완화 + 목표 비중 리밸런싱**으로 시작한다.

구현 전략명은 `liquidity_momentum_v1`이다.

## 근거

- Jegadeesh & Titman의 모멘텀 연구는 과거 승자 종목이 이후에도 상대적으로 강한 수익을 보이는 현상을 제시했다.
- Moskowitz, Ooi & Pedersen의 time-series momentum 연구는 다양한 자산군에서 최근 추세가 이후 수익 방향과 관련 있음을 제시했다.
- Barroso & Santa-Clara의 모멘텀 크래시 연구는 모멘텀 전략이 급락 구간에 취약하므로 변동성 관리가 중요하다는 점을 보여준다.
- 실전 운영에서는 거래비용과 체결 가능성이 중요하므로 거래대금/시가총액 필터를 먼저 둔다.
- 다만 거래대금과 시가총액만 강하게 반영하면 초대형주 쏠림이 생기므로, 코어 슬롯과 탐색 슬롯을 분리한다.

## 현재 구현 규칙

1. 유니버스
   - 거래량/시총 랭킹이 있으면 그 종목을 우선 사용한다.
   - `config/symbols.csv`의 관심 종목도 항상 유니버스에 포함한다.
   - 랭킹 데이터가 비어 있고 관심 종목이 대형주뿐이면 전략 후보도 대형주로만 제한된다.

2. 필터
   - 최소 가격: 1,000원
   - 최근 20거래일 평균 거래대금: 10억원 이상
   - 20일 수익률 > 0
   - 60일 수익률 > 0
   - 연율화 변동성 85% 이하

3. 점수
   - 60일 모멘텀 순위: 40%
   - 20일 모멘텀 순위: 30%
   - 평균 거래대금 순위: 10%
   - 낮은 변동성 순위: 10%
   - 대형주 편향 완화 점수: 10%

4. 포트폴리오
   - 최대 보유 종목: 5개
   - 코어 슬롯: 3개
   - 탐색 슬롯: 2개
   - 시가총액 20위 이내 초대형주는 최대 2개까지만 편입
   - 총 주식 노출: 최대 70%
   - 종목당 최대 비중: 20%
   - 현금 버퍼: 최소 10%

5. 주문
   - 기본값은 실제 주문을 실행하지 않는 dry-run이다.
   - 목표 비중과 현재 잔고를 비교해 주문 미리보기를 만든다.
   - `GOSTOP_ALLOW_LIVE_TRADING=true`와 `--confirm-live`가 둘 다 있을 때만 지정가 현금주문을 제출한다.
   - 최소 주문 금액 5만원 미만은 무시한다.
   - 시장가 주문은 사용하지 않는다.
   - 1회 주문/일일 주문 한도를 넘으면 주문을 건너뛰고 이벤트로 기록한다.

6. 하락장 리스크 오버레이
   - 수집된 종목들의 동일가중 일별 수익률로 시장 프록시를 만든다.
   - 20일 시장 프록시 수익률이 -6% 이하이거나, 60일 고점 대비 낙폭이 -10% 이하이면 `risk_off`로 보고 목표 주식 노출을 35%로 축소한다.
   - 20일 수익률이 -12% 이하이거나, 60일 고점 대비 낙폭이 -18% 이하이면 `crash`로 보고 신규 목표 노출을 0%로 낮춘다.
   - 20일 연율화 변동성이 35% 이상이면 `high_volatility`로 보고 목표 노출을 50%로 축소한다.
   - 시장 프록시 데이터가 부족하면 보수적으로 35% 노출만 허용한다.

## 운영 순서

```text
시세/일봉/랭킹/계좌 데이터 수집
→ 전략 실행
→ 시장 리스크 국면 확인
→ 전략 후보와 리밸런싱 미리보기 확인
→ dry-run 주문 기록
→ 실주문 잠금이 켜진 경우에만 지정가 주문 제출
```
